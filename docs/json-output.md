# JSON Output Reference

DevFolder emits machine-readable JSON from three subcommands. This document is the canonical reference for those shapes — useful when consuming the output from another tool (e.g. the [`devfolder-gui`](https://github.com/ahejackson/devfolder-gui) viewer).

## Commands that emit JSON

| Command | Shape | Notes |
| --- | --- | --- |
| `devfolder scan -o json` | [Scan result](#scan-result) | Tree of nodes under a root. |
| `devfolder inspect <path> -o json` | [Inspect result](#inspect-result) | Per-project git/non-git detail. |
| `devfolder report` | [Report document](#report-document) | Scan result with an inspect record embedded on every project. Always JSON. |

By default `scan` and `report` write to a file (`devfolder.json` and `report.json` respectively in the current working directory); `inspect` writes to stdout. All three accept `-f`/`--output-file` to choose the destination.

## Conventions

These rules apply across every shape below.

- **Encoding.** UTF-8 JSON, indented with two spaces.
- **Paths.** Always absolute, as strings (the OS-native form `pathlib.Path` produces).
- **Datetimes.** ISO 8601 strings produced by `datetime.isoformat()`. Timestamps generated by DevFolder itself (`generated_at`, `scanned_at`) are timezone-aware UTC. Filesystem-derived timestamps (`mtime`, `last_commit_at`) carry whatever timezone the source provides.
- **Nullability.** Fields documented with type `T | null` may be `null`; everything else is always present.
- **Discriminators.** Tagged unions use a `kind` field. Two unions exist: [node kinds](#node-kinds) (`project`, `category`, `symlink`, `ignored`, `error`) and [inspect kinds](#inspect-result) (`git`, `non-git`).

## Scan result

The top-level object produced by `scan` (and the base of a [report document](#report-document)).

| Field | Type | Description |
| --- | --- | --- |
| `generated_at` | string | When the scan started (UTC, ISO 8601). |
| `root` | string | Absolute path of the directory that was scanned. |
| `is_root_project` | boolean | `true` when the root itself is a project (a `.git` directory was found at `root`). |
| `root_project` | [Project node](#project-node) \| null | The project record for `root` when `is_root_project` is `true`; otherwise `null`. |
| `children` | array of [Node](#node-kinds) | Direct children of `root`. Empty when `is_root_project` is `true` (no further descent). |

### Example

```json
{
  "generated_at": "2026-04-27T10:15:30.123456+00:00",
  "root": "/Users/adam/Dev",
  "is_root_project": false,
  "root_project": null,
  "children": [
    {
      "kind": "category",
      "name": "tools",
      "path": "/Users/adam/Dev/tools",
      "is_empty": false,
      "children": [
        {
          "kind": "project",
          "name": "devfolder-python",
          "path": "/Users/adam/Dev/tools/devfolder-python",
          "project_type": "owned-remote",
          "remote_url": "git@github.com:ahejackson/devfolder-python.git",
          "owner": "ahejackson"
        }
      ]
    },
    {
      "kind": "ignored",
      "name": ".config",
      "path": "/Users/adam/Dev/.config",
      "reason": "dotfolder"
    }
  ]
}
```

## Node kinds

Every entry in `children` (and `root_project`, when present) is a node. Discriminate on `kind`. All nodes share `kind`, `name`, and `path`.

### Project node

A folder containing `.git`, or a leaf folder at depth 2.

| Field | Type | Description |
| --- | --- | --- |
| `kind` | `"project"` | Discriminator. |
| `name` | string | Folder name. |
| `path` | string | Absolute path. |
| `project_type` | [Project type](#project-types) | Classification. |
| `remote_url` | string \| null | First matched remote URL when `project_type` is `owned-remote` or `other-remote`; otherwise `null`. |
| `owner` | string \| null | Configured owner name when `project_type` is `owned-remote`; otherwise `null`. |
| `inspect` | [Inspect result](#inspect-result) \| null | **Only present in [report documents](#report-document).** The per-project inspect record. |

### Category node

A folder at depth 1 that is not itself a project. Categories are the only nodes with children.

| Field | Type | Description |
| --- | --- | --- |
| `kind` | `"category"` | Discriminator. |
| `name` | string | Folder name. |
| `path` | string | Absolute path. |
| `is_empty` | boolean | `true` when `children` is empty. |
| `children` | array of [Node](#node-kinds) | Nested nodes. Typically projects, symlinks, ignored entries, and error nodes — categories do not nest within categories. |

### Symlink node

A symbolic link. Symlinks are recorded but not followed.

| Field | Type | Description |
| --- | --- | --- |
| `kind` | `"symlink"` | Discriminator. |
| `name` | string | Link name. |
| `path` | string | Absolute path of the link itself. |
| `target` | string \| null | The link target as recorded on disk. `null` when the target couldn't be read. |

### Ignored node

A folder skipped by the scanner.

| Field | Type | Description |
| --- | --- | --- |
| `kind` | `"ignored"` | Discriminator. |
| `name` | string | Folder name. |
| `path` | string | Absolute path. |
| `reason` | `"dotfolder"` \| `"node_modules"` | Why the folder was skipped. |

### Error node

A folder that could not be scanned.

| Field | Type | Description |
| --- | --- | --- |
| `kind` | `"error"` | Discriminator. |
| `name` | string | Folder name. |
| `path` | string | Absolute path. |
| `error_message` | string | Human-readable description of the failure (typically a permission error). May be empty. |

## Project types

The `project_type` field on a project node is one of:

| Value | Meaning |
| --- | --- |
| `empty` | Project folder with no contents. |
| `local-untracked` | No `.git` folder. |
| `local-git` | Has `.git` but no remotes configured. |
| `owned-remote` | At least one remote matches a configured `(host, name)` owner. |
| `other-remote` | Has remotes, but none match a configured owner. |

See [the project README](../README.md#configuration) for owner configuration.

## Inspect result

The top-level object produced by `inspect`. Tagged on `kind`: `"git"` for projects with a `.git` directory, `"non-git"` for empty or untracked folders.

### Git inspect (`kind: "git"`)

| Field | Type | Description |
| --- | --- | --- |
| `kind` | `"git"` | Discriminator. |
| `path` | string | Absolute path of the project. |
| `working_tree` | object | See [Working tree](#working-tree). |
| `branches` | object | See [Branches](#branches). |
| `stash_count` | integer | Number of stash entries. |
| `last_commit_at` | string \| null | ISO 8601 timestamp of the most recent commit on any branch. `null` when the repo has no commits. |
| `mtime` | string | ISO 8601 mtime of the project directory. |
| `remotes` | array of [Remote](#remote) | All configured git remotes. May be empty. |
| `scanned_at` | string | When the inspect ran (UTC, ISO 8601). |

#### Working tree

| Field | Type | Description |
| --- | --- | --- |
| `clean` | boolean | `true` when there are no staged, modified, or untracked files. |
| `staged` | integer | Count of staged changes. |
| `modified` | integer | Count of unstaged modifications to tracked files. |
| `untracked` | integer | Count of untracked files. |

#### Branches

| Field | Type | Description |
| --- | --- | --- |
| `total` | integer | Total local branches. |
| `no_upstream` | integer | Local branches with no configured upstream. |
| `ahead_of_upstream` | integer | Local branches whose tip is ahead of their upstream. |

#### Remote

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | The remote name (e.g. `origin`). |
| `url` | string | The remote URL as stored by git. |
| `host` | string \| null | Parsed host (e.g. `github.com`). `null` when the URL shape is unfamiliar. |
| `owner` | string \| null | Parsed owner (e.g. `ahejackson`). `null` when the URL shape is unfamiliar. |
| `repo` | string \| null | Parsed repo name. `null` when the URL shape is unfamiliar. |

#### Example

```json
{
  "kind": "git",
  "path": "/Users/adam/Dev/tools/devfolder-python",
  "working_tree": {
    "clean": false,
    "staged": 0,
    "modified": 1,
    "untracked": 2
  },
  "branches": {
    "total": 3,
    "no_upstream": 1,
    "ahead_of_upstream": 1
  },
  "stash_count": 0,
  "last_commit_at": "2026-04-26T18:42:11+01:00",
  "mtime": "2026-04-27T09:55:02+01:00",
  "remotes": [
    {
      "name": "origin",
      "url": "git@github.com:ahejackson/devfolder-python.git",
      "host": "github.com",
      "owner": "ahejackson",
      "repo": "devfolder-python"
    }
  ],
  "scanned_at": "2026-04-27T10:15:31.987654+00:00"
}
```

### Non-git inspect (`kind: "non-git"`)

Produced for projects classified as `empty` or `local-untracked`.

| Field | Type | Description |
| --- | --- | --- |
| `kind` | `"non-git"` | Discriminator. |
| `path` | string | Absolute path of the project. |
| `file_count` | integer | Total files under `path` (recursive). |
| `folder_count` | integer | Total folders under `path` (recursive). |
| `total_size_bytes` | integer | Sum of file sizes in bytes. |
| `mtime` | string | ISO 8601 mtime of the project directory. |
| `scanned_at` | string | When the inspect ran (UTC, ISO 8601). |

#### Example

```json
{
  "kind": "non-git",
  "path": "/Users/adam/Dev/scratch/half-finished-idea",
  "file_count": 12,
  "folder_count": 3,
  "total_size_bytes": 48211,
  "mtime": "2025-11-04T22:13:40+00:00",
  "scanned_at": "2026-04-27T10:15:32.112233+00:00"
}
```

## Report document

The output of `devfolder report` is a [scan result](#scan-result) with one addition: every [project node](#project-node) gains an `inspect` field containing its [inspect result](#inspect-result). All other node kinds are unchanged.

`inspect` is typed `Inspect | null`. In practice `report` inspects every project, so the field is populated for all projects produced by the current implementation. Consumers should still handle `null` defensively to stay forward-compatible.

### Example fragment

```json
{
  "kind": "project",
  "name": "devfolder-python",
  "path": "/Users/adam/Dev/tools/devfolder-python",
  "project_type": "owned-remote",
  "remote_url": "git@github.com:ahejackson/devfolder-python.git",
  "owner": "ahejackson",
  "inspect": {
    "kind": "git",
    "path": "/Users/adam/Dev/tools/devfolder-python",
    "working_tree": { "clean": true, "staged": 0, "modified": 0, "untracked": 0 },
    "branches": { "total": 1, "no_upstream": 0, "ahead_of_upstream": 0 },
    "stash_count": 0,
    "last_commit_at": "2026-04-26T18:42:11+01:00",
    "mtime": "2026-04-27T09:55:02+01:00",
    "remotes": [
      {
        "name": "origin",
        "url": "git@github.com:ahejackson/devfolder-python.git",
        "host": "github.com",
        "owner": "ahejackson",
        "repo": "devfolder-python"
      }
    ],
    "scanned_at": "2026-04-27T10:15:31.987654+00:00"
  }
}
```

## Versioning

This format is pre-1.0 alongside DevFolder itself. Breaking changes ship in minor releases (see [`CHANGELOG.md`](../CHANGELOG.md)). When the schema stabilises and a `1.0` is cut, breaking changes will move to major bumps.
